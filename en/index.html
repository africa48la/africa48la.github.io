What text encoding does (and does not) do

Base64, URL-escaping and Hex are encodings: they rewrite bytes into a safe character set so text and binary data survive transport through email, URLs and JSON. They are fully reversible by anyone and provide no secrecy. Use them to move data safely, not to hide it.

How Base64 works

Base64 is defined by RFC 4648. It groups the input into 6-bit chunks and maps each chunk to one of 64 characters (A-Z, a-z, 0-9, plus + and /), padding with = at the end. Three input bytes become four output characters, so the encoded text is about 33% larger than the input. The URL-safe variant replaces + and / with - and _ so the result is safe inside links.

Encoding is not encryption

A common mistake is to treat Base64 as a way to protect data. It is not: anyone can decode it. The passphrase cipher in this tool adds a key, but a simple XOR is only light obfuscation and is not secure against a determined attacker. For real confidentiality, use a messenger with proper end-to-end encryption, such as Telegram Secret Chats, which use the MTProto protocol with 256-bit AES.

FAQ

Is my text uploaded anywhere?

No. All encoding, decoding and the cipher run in your browser with standard web APIs; your text never leaves your device.

Does Base64 encrypt my data?

No. Base64 is an encoding, not encryption - anyone can decode it. It only makes bytes safe to transport, it does not keep them secret.

Why does Base64 of emoji or Chinese look different from other tools?

Because this tool encodes the correct UTF-8 bytes via TextEncoder. Tools that call btoa() directly fail or corrupt characters above Latin1.

What is the difference between URL encoding and Base64?

URL encoding (encodeURIComponent) escapes characters that are unsafe in a URL using % sequences; Base64 rewrites raw bytes into a 64-character alphabet. They solve different transport problems.

Is the passphrase cipher secure?

No. It is a simple XOR for light obfuscation only. For real confidentiality use a proper end-to-end encrypted messenger such as Telegram Secret Chats.

What is the = at the end of Base64?

It is padding. Base64 works in blocks of four characters; = fills the last block when the input length is not a multiple of three bytes.
