ตัวเข้ารหัส Base64 / URL / Hex และการเข้ารหัสอย่างง่าย

วางข้อความใดๆ เพื่อเข้ารหัสหรือถอดรหัสเป็น Base64, URL-escape หรือ Hex และเพื่อนับไบต์และอักขระ UTF-8 นอกจากนี้ยังมีการเข้ารหัสข้อความรหัสผ่านอย่างง่ายเพื่อการสร้างความสับสนอย่างรวดเร็ว ทุกอย่างทำงานในเบราว์เซอร์ของคุณ ข้อความของคุณไม่เคยถูกอัปโหลด

เข้ารหัส ถอดรหัส และนับ

ทำงานได้อย่างสมบูรณ์ในเบราว์เซอร์ของคุณ Base64 ใช้ TextEncoder ดังนั้น Unicode และอิโมจิจึงได้รับการจัดการอย่างถูกต้อง

0 UTF-8 ไบต์ · 0 ตัวละคร

การสร้างความสับสนอย่างง่ายด้วย XOR - ไม่สามารถแทนที่การเข้ารหัสจากต้นทางถึงปลายทางได้จริง

การเข้ารหัสข้อความทำอะไร (และไม่) ทำ

Base64, URL-escaping และ Hex เป็นการเข้ารหัส โดยจะเขียนไบต์ใหม่เป็นชุดอักขระที่ปลอดภัย เพื่อให้ข้อความและข้อมูลไบนารีสามารถขนส่งผ่านอีเมล URL และ JSON ได้ ทุกคนสามารถย้อนกลับได้อย่างสมบูรณ์และไม่มีการเก็บความลับ ใช้เพื่อย้ายข้อมูลอย่างปลอดภัย ไม่ใช่เพื่อซ่อนข้อมูล

Base64 ทำงานอย่างไร

Base64 ถูกกำหนดโดย RFC 4648 โดยจะจัดกลุ่มอินพุตออกเป็นชิ้นขนาด 6 บิต และแมปแต่ละชิ้นให้มีอักขระ 64 ตัว (A-Z, a-z, 0-9, บวก + และ /) โดยเติมด้วย = ต่อท้าย ไบต์อินพุตสามไบต์กลายเป็นอักขระเอาต์พุตสี่ตัว ดังนั้นข้อความที่เข้ารหัสจึงมีขนาดใหญ่กว่าอินพุตประมาณ 33% รูปแบบที่ปลอดภัยของ URL จะแทนที่ + และ / ด้วย - และ _ ดังนั้นผลลัพธ์จึงปลอดภัยภายในลิงก์

การเข้ารหัสไม่ใช่การเข้ารหัส

ข้อผิดพลาดทั่วไปคือถือว่า Base64 เป็นวิธีการปกป้องข้อมูล ไม่ใช่: ใครๆ ก็สามารถถอดรหัสได้ การเข้ารหัสข้อความรหัสผ่านในเครื่องมือนี้จะเพิ่มคีย์ แต่ XOR แบบธรรมดานั้นเป็นเพียงการสร้างความสับสนเล็กน้อย และไม่ปลอดภัยต่อผู้โจมตีที่กำหนด เพื่อการรักษาความลับอย่างแท้จริง ให้ใช้ Messenger ที่มีการเข้ารหัสจากต้นทางถึงปลายทางที่เหมาะสม เช่น Telegram Secret Chats ซึ่งใช้โปรโตคอล MTProto พร้อมด้วย AES 256 บิต

การเข้ารหัสค่าคงที่และการเข้ารหัสข้อความ (สแน็ปช็อต 2026)

ค่าการเข้ารหัสด้านล่างนี้เป็นข้อเท็จจริงที่แน่นอนจาก RFC 4648 และ MDN ตัวเลขการส่งข้อความเป็นการประมาณการปี 2026 ที่อนุมานจากข้อมูลพื้นฐานสาธารณะ (ตัวเลขที่ประกาศของ Telegram และ Wikipedia) ไม่ใช่การวัดผลโดยตรง

การเข้ารหัสข้อความทำอะไรการเข้ารหัสค่าคงที่และกา
Base64 ขนาดเหนือศีรษะ+33% (อินพุตสามไบต์กลายเป็นอักขระสี่ตัว) ต่อ RFC 4648 - แน่นอน
ตัวอักษร Base6464 ตัวอักษร: A-Z, a-z, 0-9, + และ /; ช่องว่างภายใน =; การใช้รูปแบบที่ปลอดภัยของ URL - และ _ (RFC 4648 §4-5) - ทุกประการ
btoa / atob ขีดจำกัดใช้งานได้กับ Latin1 เท่านั้น UTF-8 และอีโมจิต้องมี TextEncoder / TextDecoder (MDN) - แน่นอน
Telegram ผู้ใช้ที่ใช้งานรายเดือนทะลุ 1 พันล้านในเดือนมีนาคม 2568 (Pavel Durov) ประมาณการปี 2569: เติบโตอย่างต่อเนื่อง
การเข้ารหัส TelegramMTProto พร้อม AES 256 บิต, RSA 2048 บิต และ Diffie-Hellman; จากต้นทางถึงปลายทางเฉพาะใน Secret Chat ไม่ใช่ค่าเริ่มต้น - ตาม Wikipedia
นำไปใช้ประโยชน์ได้จริงใช้การเข้ารหัสเพื่อย้ายข้อความอย่างปลอดภัย ใช้ Secret Chats (หรือแอป end-to-end อื่น) เมื่อคุณต้องการความลับ

ค่าคงที่การเข้ารหัสนั้นแน่นอน (RFC 4648 / MDN) การใช้การรับส่งข้อความเป็นค่าประมาณในปี 2026 ที่คาดการณ์จากข้อมูลพื้นฐานสาธารณะที่อ้างถึงและการเปลี่ยนแปลงเมื่อเวลาผ่านไป

FAQ

ข้อความของฉันถูกอัพโหลดทุกที่หรือไม่?

ไม่ การเข้ารหัส การถอดรหัส และการเข้ารหัสทั้งหมดทำงานในเบราว์เซอร์ของคุณด้วย API เว็บมาตรฐาน ข้อความของคุณจะไม่ออกจากอุปกรณ์ของคุณ

Base64 เข้ารหัสข้อมูลของฉันหรือไม่

ไม่ Base64 เป็นการเข้ารหัส ไม่ใช่การเข้ารหัส ใครๆ ก็สามารถถอดรหัสได้ มันทำให้ไบต์ปลอดภัยในการขนส่งเท่านั้น มันไม่ได้เก็บเป็นความลับ

เหตุใด Base64 ของอิโมจิหรือภาษาจีนจึงดูแตกต่างจากเครื่องมืออื่น ๆ

เนื่องจากเครื่องมือนี้เข้ารหัสไบต์ UTF-8 ที่ถูกต้องผ่าน TextEncoder เครื่องมือที่เรียก btoa() ล้มเหลวโดยตรงหรือเสียหายอักขระที่อยู่เหนือ Latin1

การเข้ารหัส URL และ Base64 แตกต่างกันอย่างไร

การเข้ารหัส URL (encodeURIComponent) จะหลีกอักขระที่ไม่ปลอดภัยใน URL โดยใช้ลำดับ % Base64 เขียนไบต์ดิบใหม่เป็นตัวอักษร 64 ตัว พวกเขาแก้ปัญหาการขนส่งที่แตกต่างกัน

รหัสข้อความรหัสผ่านมีความปลอดภัยหรือไม่?

ไม่ มันเป็น XOR ธรรมดาสำหรับการสร้างความสับสนเล็กน้อยเท่านั้น สำหรับการรักษาความลับที่แท้จริง ให้ใช้โปรแกรมรับส่งข้อความที่เข้ารหัสจากต้นทางถึงปลายทางที่เหมาะสม เช่น Telegram Secret Chats

= ท้าย Base64 คืออะไร?

มันเป็นการบุนวม Base64 ทำงานเป็นบล็อกสี่อักขระ = เติมบล็อกสุดท้ายเมื่อความยาวอินพุตไม่เป็นผลคูณของสามไบต์

รับ Telegram

Telegram เป็นแอปส่งข้อความที่เข้ารหัสซึ่งมี Secret Chat สำหรับการเข้ารหัสจากต้นทางถึงปลายทาง ดาวน์โหลดสำหรับแพลตฟอร์มของคุณ:

Telegram - ดาวน์โหลดทุกแพลตฟอร์ม

อ้างอิง

  • Base64 - วิกิพีเดีย (RFC 4648)
  • MDN - Base64 ใน JavaScript
  • Telegram - วิกิพีเดีย
  • MDN - encodeURIComponent
