ຕົວເຂົ້າລະຫັດ Base64 / URL / Hex ແລະລະຫັດລັບແບບງ່າຍໆ

ວາງຂໍ້ຄວາມໃດນຶ່ງເພື່ອເຂົ້າລະຫັດ ຫຼືຖອດລະຫັດມັນເປັນ Base64, URL-escaped, ຫຼື Hex, ແລະເພື່ອນັບ UTF-8 bytes ແລະຕົວອັກສອນຂອງມັນ. ນອກນັ້ນຍັງມີລະຫັດລັບຂອງລະຫັດຜ່ານແບບງ່າຍດາຍສໍາລັບການເຮັດໃຫ້ສັບສົນໄວ. ທຸກສິ່ງທຸກຢ່າງເຮັດວຽກຢູ່ໃນຕົວທ່ອງເວັບຂອງທ່ານ; ຂໍ້ຄວາມຂອງເຈົ້າບໍ່ເຄີຍຖືກອັບໂຫຼດ.

ເຂົ້າລະຫັດ, ຖອດລະຫັດ ແລະນັບ

ດໍາເນີນການທັງຫມົດໃນຕົວທ່ອງເວັບຂອງທ່ານ. Base64 ໃຊ້ TextEncoder ເພື່ອໃຫ້ Unicode ແລະ emoji ຖືກຈັດການຢ່າງຖືກຕ້ອງ.

0 UTF-8 ໄບຕ໌ · 0 ຕົວອັກສອນ

XOR obfuscation ງ່າຍດາຍ - ບໍ່ແມ່ນການທົດແທນສໍາລັບການເຂົ້າລະຫັດ end-to-end ທີ່ແທ້ຈິງ.

ການເຂົ້າລະຫັດຂໍ້ຄວາມແມ່ນຫຍັງ (ແລະບໍ່) ເຮັດ

Base64, URL-escaping ແລະ Hex ແມ່ນການເຂົ້າລະຫັດ: ພວກເຂົາຂຽນຄືນໃຫມ່ bytes ເຂົ້າໄປໃນຊຸດຕົວອັກສອນທີ່ປອດໄພເພື່ອໃຫ້ຂໍ້ຄວາມແລະຂໍ້ມູນຖານສອງຢູ່ລອດການຂົນສົ່ງຜ່ານອີເມວ, URLs ແລະ JSON. ພວກມັນສາມາດປີ້ນກັບກັນໄດ້ຢ່າງເຕັມສ່ວນໂດຍໃຜແລະບໍ່ໃຫ້ຄວາມລັບ. ໃຊ້ພວກມັນເພື່ອຍ້າຍຂໍ້ມູນຢ່າງປອດໄພ, ບໍ່ແມ່ນເພື່ອເຊື່ອງມັນ.

Base64 ເຮັດວຽກແນວໃດ

Base64 ຖືກກໍານົດໂດຍ RFC 4648. ມັນຈັດກຸ່ມການປ້ອນເຂົ້າເປັນ chunks 6-bit ແລະແຜນທີ່ແຕ່ລະ chunk ເປັນຫນຶ່ງຂອງ 64 ຕົວອັກສອນ (A-Z, a-z, 0-9, ບວກ + ແລະ /), padding ກັບ = ໃນຕອນທ້າຍຂອງ. ສາມ input bytes ກາຍ​ເປັນ​ສີ່​ຕົວ​ອັກ​ສອນ​ອອກ​, ດັ່ງ​ນັ້ນ​ຂໍ້​ຄວາມ​ທີ່​ເຂົ້າ​ລະ​ຫັດ​ແມ່ນ​ປະ​ມານ 33​% ຂະ​ຫນາດ​ໃຫຍ່​ກ​່​ວາ​ການ​ປ້ອນ​ຂໍ້​ມູນ​. ຕົວປ່ຽນແປງ URL-safe ແທນ + ແລະ / ກັບ - ແລະ _ ດັ່ງນັ້ນຜົນໄດ້ຮັບແມ່ນປອດໄພພາຍໃນການເຊື່ອມຕໍ່.

ການເຂົ້າລະຫັດບໍ່ແມ່ນການເຂົ້າລະຫັດ

ຄວາມຜິດພາດທົ່ວໄປແມ່ນການປິ່ນປົວ Base64 ເປັນວິທີການປົກປ້ອງຂໍ້ມູນ. ມັນບໍ່ແມ່ນ: ໃຜສາມາດຖອດລະຫັດມັນໄດ້. ລະຫັດລະຫັດຜ່ານໃນເຄື່ອງມືນີ້ເພີ່ມກະແຈ, ແຕ່ XOR ງ່າຍໆພຽງແຕ່ເປັນການລົບກວນທີ່ເບົາບາງ ແລະ ບໍ່ປອດໄພຕໍ່ກັບຜູ້ໂຈມຕີທີ່ກຳນົດໄວ້. ສໍາລັບຄວາມລັບທີ່ແທ້ຈິງ, ໃຫ້ໃຊ້ messenger ທີ່ມີການເຂົ້າລະຫັດແບບຕົ້ນທາງເຖິງຕອນທ້າຍທີ່ເຫມາະສົມ, ເຊັ່ນ Telegram Secret Chats, ເຊິ່ງໃຊ້ໂປໂຕຄອນ MTProto ກັບ 256-bit AES.

ການເຂົ້າລະຫັດຄົງທີ່ & ການເຂົ້າລະຫັດຂໍ້ຄວາມ (ຮູບຖ່າຍ 2026)

ຄ່າການເຂົ້າລະຫັດຂ້າງລຸ່ມນີ້ແມ່ນຄວາມຈິງທີ່ແນ່ນອນຈາກ RFC 4648 ແລະ MDN; ຕົວ​ເລກ​ການ​ສົ່ງ​ຂໍ້​ຄວາມ​ແມ່ນ​ການ​ຄາດ​ຄະ​ເນ​ໃນ​ປີ 2026 ທີ່​ໄດ້​ຍົກ​ອອກ​ມາ​ຈາກ​ການ​ຄາດ​ຄະ​ເນ​ຂອງ​ສາ​ທາ​ລະ​ນະ (ຕົວ​ເລກ​ການ​ປະ​ກາດ Telegram ແລະ Wikipedia​) - ບໍ່​ແມ່ນ​ການ​ວັດ​ແທກ​ດ້ວຍ​ມື​ທໍາ​ອິດ​.

ຕົວອັກສອນ Base6464 ຕົວອັກສອນ: A-Z, a-z, 0-9, + ແລະ /; padding = ; ໃຊ້ຕົວແປທີ່ປອດໄພ URL - ແລະ _ (RFC 4648 §4-5) - ຄືກັນອ້ອຍຕ້ອຍ

ການເຂົ້າລະຫັດຄົງທີ່ແນ່ນອນ (RFC 4648 / MDN); ການຮັບຮອງເອົາການສົ່ງຂໍ້ຄວາມແມ່ນການຄາດຄະເນຂອງປີ 2026 ທີ່ອ້າງອີງຈາກພື້ນຖານສາທາລະນະທີ່ອ້າງອີງແລະການປ່ຽນແປງໃນໄລຍະເວລາ.

FAQ

ຂໍ້ຄວາມຂອງຂ້ອຍຖືກອັບໂຫລດຢູ່ບ່ອນໃດ?

ບໍ່. ການເຂົ້າລະຫັດ, ການຖອດລະຫັດ ແລະລະຫັດລັບທັງໝົດດຳເນີນຢູ່ໃນບຣາວເຊີຂອງທ່ານດ້ວຍ API ຂອງເວັບມາດຕະຖານ; ຂໍ້ຄວາມຂອງເຈົ້າບໍ່ເຄີຍອອກຈາກອຸປະກອນຂອງທ່ານ.

Base64 ເຂົ້າລະຫັດຂໍ້ມູນຂອງຂ້ອຍບໍ?

ສະບັບເລກທີ Base64 ແມ່ນການເຂົ້າລະຫັດ, ບໍ່ແມ່ນການເຂົ້າລະຫັດ - ທຸກຄົນສາມາດຖອດລະຫັດມັນໄດ້. ມັນພຽງແຕ່ເຮັດໃຫ້ bytes ປອດໄພໃນການຂົນສົ່ງ, ມັນບໍ່ໄດ້ເກັບຮັກສາໃຫ້ເຂົາເຈົ້າເປັນຄວາມລັບ.

ເປັນຫຍັງ Base64 ຂອງ emoji ຫຼືຈີນເບິ່ງແຕກຕ່າງຈາກເຄື່ອງມືອື່ນໆ?

ເນື່ອງຈາກວ່າເຄື່ອງມືນີ້ເຂົ້າລະຫັດ UTF-8 bytes ທີ່ຖືກຕ້ອງຜ່ານ TextEncoder. ເຄື່ອງມືທີ່ໂທຫາ btoa() ລົ້ມເຫລວໂດຍກົງຫຼືຕົວລະຄອນທີ່ເສຍຫາຍຂ້າງເທິງ Latin1.

ຄວາມແຕກຕ່າງລະຫວ່າງການເຂົ້າລະຫັດ URL ແລະ Base64 ແມ່ນຫຍັງ?

ການເຂົ້າລະຫັດ URL (encodeURIComponent) ໜີອອກຈາກຕົວອັກສອນທີ່ບໍ່ປອດໄພໃນ URL ໂດຍໃຊ້ % ລຳດັບ; Base64 ຂຽນຄືນໄບຕ໌ດິບເປັນຕົວອັກສອນ 64 ຕົວອັກສອນ. ພວກເຂົາແກ້ໄຂບັນຫາການຂົນສົ່ງທີ່ແຕກຕ່າງກັນ.

ລະຫັດລັບຂອງລະຫັດຜ່ານປອດໄພບໍ?

ບໍ່. ມັນເປັນ XOR ງ່າຍດາຍສໍາລັບການ obfuscation ແສງສະຫວ່າງເທົ່ານັ້ນ. ເພື່ອຄວາມລັບທີ່ແທ້ຈິງໃຫ້ໃຊ້ຕົວສົ່ງຂ່າວທີ່ເຂົ້າລະຫັດແບບຕົ້ນທາງເຖິງຈຸດຈົບທີ່ເຫມາະສົມເຊັ່ນ Telegram Secret Chats.

ແມ່ນຫຍັງ = ໃນຕອນທ້າຍຂອງ Base64?

ມັນແມ່ນ padding. Base64 ເຮັດວຽກຢູ່ໃນຕັນຂອງສີ່ຕົວອັກສອນ; = ຕື່ມຂໍ້ມູນໃສ່ບລັອກສຸດທ້າຍເມື່ອຄວາມຍາວການປ້ອນຂໍ້ມູນບໍ່ແມ່ນຫຼາຍຂອງສາມໄບຕ໌.

ເອົາ Telegram

Telegram ເປັນແອັບສົ່ງຂໍ້ຄວາມທີ່ຖືກເຂົ້າລະຫັດດ້ວຍ Secret Chats ສຳລັບການເຂົ້າລະຫັດແບບຕົ້ນທາງ. ດາວນ໌ໂຫລດມັນສໍາລັບເວທີຂອງທ່ານ:
