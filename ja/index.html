Base64 / URL / Hex エンコーダー & 簡易暗号

任意のテキストを貼り付ければ、Base64・URL エスケープ・Hex にエンコード/デコードでき、UTF-8 のバイト数と文字数も数えられます。さらに手早い難読化のための簡易パスフレーズ暗号もあります。すべてブラウザ内で動作し、テキストはアップロードされません。

エンコード・デコード・カウント

すべてブラウザ内で実行。Base64 は TextEncoder を使うため Unicode と絵文字も正しく扱えます。

文字エンコードができること・できないこと

Base64・URL エスケープ・Hex はエンコードです。バイトを安全な文字集合に書き換え、テキストやバイナリをメール・URL・JSON 経由で安全に運べるようにします。誰でも完全に元へ戻せ、秘匿性はありません。データを隠すためではなく、安全に運ぶために使ってください。

Base64 の仕組み

Base64 は RFC 4648 で定義されています。入力を 6 ビットごとに分け、各塊を 64 文字（A–Z、a–z、0–9 と + /）のいずれかに対応させ、末尾を = で詰めます。3 つの入力バイトが 4 つの出力文字になるため、エンコード後は入力より約 33% 大きくなります。URL セーフ版は + と / を - と _ に置き換え、リンク内でも安全にします。

エンコードは暗号化ではない

Base64 をデータ保護の手段と思うのはよくある誤解です。違います。誰でもデコードできます。本ツールのパスフレーズ暗号は鍵を加えますが、単純な XOR は軽い難読化にすぎず、本気の攻撃者には通用しません。真に秘匿したいなら、Telegram の Secret Chat のような真の端到端暗号を持つメッセンジャーを使ってください。MTProto と 256 ビット AES を採用しています。

FAQ

私のテキストはアップロードされますか？

いいえ。エンコード・デコード・暗号はすべて標準 Web API でブラウザ内処理され、テキストは端末から出ません。

Base64 はデータを暗号化しますか？

いいえ。Base64 は暗号化ではなくエンコードで、誰でもデコードできます。運搬を安全にするだけで秘匿はしません。

絵文字や日本語の Base64 が他ツールと違うのはなぜ？

本ツールは TextEncoder で正しい UTF-8 バイトをエンコードするためです。btoa() を直接呼ぶツールは Latin1 を超える文字で失敗・破損します。

URL エンコードと Base64 の違いは？

URL エンコード（encodeURIComponent）は URL で危険な文字を % 列でエスケープし、Base64 は生バイトを 64 文字集合に書き換えます。解決する問題が異なります。

このパスフレーズ暗号は安全ですか？

いいえ。軽い難読化のための単純な XOR です。真に秘匿したいなら Telegram Secret Chat のような真の端到端暗号を使ってください。

Base64 末尾の = は何ですか？

パディングです。Base64 は 4 文字ブロック単位で動き、入力長が 3 バイトの倍数でないとき = で最後のブロックを埋めます。
